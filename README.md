# Разделитель текстовых файлов (TXT File Splitter)

[![Русский](https://img.shields.io/badge/lang-%D0%A0%D1%83%D1%81%D1%81%D0%BA%D0%B8%D0%B9-blue.svg)](README.md)
[![English](https://img.shields.io/badge/lang-English-lightgrey.svg)](README.en.md)

Графическое приложение для разрезания больших текстовых файлов на несколько частей
по заданному предельному размеру. Написано на Python с использованием Tkinter,
работает без установки дополнительных библиотек.

---

## Возможности

- 🖱 **Графический интерфейс** — выбор файла и папки через диалоги, журнал выполнения, индикатор прогресса.
- ✂️ **Три способа разделения** — по размеру, по строкам или по пользовательскому разделителю.
- 🔎 **Автоопределение кодировки** — UTF-8 (в т.ч. с BOM), CP1251, Latin-1.
- ⚙️ **Фоновое выполнение** — интерфейс не зависает во время работы.
- 📂 **Открытие папки результата** в системном файловом менеджере одним кликом.
- 🍎 **Запуск двойным кликом** на macOS через файл `start.command`.

---

## Способы разделения

| Способ | Описание |
| --- | --- |
| **Просто по размеру** | Файл режется ровно по байтам. Кодировка не важна, но многобайтовые символы могут оказаться разрезанными — это ожидаемо для данного режима. |
| **По переносу строки** | Границы частей никогда не проходят внутри строки. Строки целиком помещаются в файл, пока не будет достигнут предельный размер. |
| **По набору символов** | Разделение по заданному разделителю (по умолчанию `---`). Сегменты вместе с разделителем группируются в файлы, не превышая лимита. Одиночный сегмент, превышающий лимит, записывается целиком. |

---

## Имена файлов результатов

Части сохраняются в выбранную папку с именами вида:

```
{имя_файла}_part001.txt
{имя_файла}_part002.txt
{имя_файла}_part003.txt
...
```

Порядковый номер дополняется нулями до трёх знаков, исходное расширение сохраняется.

## Работа с кодировками

Приложение автоматически определяет кодировку файла (по первому мегабайту):

1. **UTF-8 с BOM** — читается через `utf-8-sig` (BOM удаляется), записывается обычным `utf-8`, чтобы BOM не дублировался в каждой части.
2. **UTF-8 / CP1251 / Latin-1** — определяются по возможности декодирования.
3. В крайнем случае используется `Latin-1` (никогда не вызывает ошибок).

Для режима «Просто по размеру» кодировка не определяется — файл обрабатывается в бинарном режиме.

## Запуск

### macOS — двойной клик

1. Откройте папку проекта в Finder.
2. Дважды кликните по файлу **`start.command`**.
3. При первом запуске разрешите открытие в *Системные настройки → Конфиденциальность и безопасность*.

> Файл `start.command` проверяет наличие `python3` и при его отсутствии показывает
> понятное сообщение с подсказкой об установке.

### Из командной строки (любая ОС)

```bash
python3 txt_file_splitter.py
```

## Использование

1. **Исходный файл** — нажмите «Обзор…» и выберите текстовый файл для разделения.
2. **Папка для сохранения** — укажите, куда сохранить части (по умолчанию предлагается папка рядом с исходным файлом).
3. **Предельный размер** — задайте максимальный размер одной части и выберите единицу измерения (Байт / КБ / МБ). Значения — в двоичных единицах (1 КБ = 1024 байт, 1 МБ = 1024 КБ).
4. **Способ разделения** — выберите один из трёх режимов (см. таблицу выше).
5. **Разделитель** *(только для режима «По набору символов»)* — укажите строку-разделитель.
6. Нажмите **«Разделить»** — в журнале отобразится прогресс и количество созданных частей.
7. Кнопка **«Открыть папку результата»** покажет готовые файлы в файловом менеджере.
